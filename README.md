# Proxy Scraper — сборщик и валидатор прокси-серверов

[![Python Version](https://img.shields.io/badge/python-3.8%2B-blue.svg)](https://python.org)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

**Proxy Scraper** — это инструмент для автоматического сбора, проверки и экспорта рабочих прокси-конфигов (VLESS, Trojan и другие) из множества открытых источников. Скрипт скачивает списки URL, проверяет каждый прокси на доступность, измеряет задержки (TCP/HTTP) и сохраняет историю проверок в структурированном виде. На выходе вы получаете готовые ссылки для использования в VPN-клиентах (например, V2Ray, Xray).

## Возможности

- ✅ **Сбор** — загружает списки прокси из указанных источников (см. `input.txt`).
- ✅ **Проверка** — тестирует каждый прокси на работоспособность, измеряет TCP и HTTP задержки.
- ✅ **История** — сохраняет результаты всех проверок в `history.json` с метриками качества (score, latency, uptime).
- ✅ **Экспорт** — генерирует файл `output.txt` с готовыми VLESS-ссылками для быстрого импорта в клиенты.
- ✅ **Статистика** — ведёт статистику по источникам (`source_stats.json`) и топ-30 лучших прокси (`top30.txt`).

## Установка и запуск

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/LexterS999/proxy_scraper.git
cd proxy_scraper
```

### 2. Установите зависимости

```
pip install -r requirements.txt
```

> Рекомендуется использовать виртуальное окружение (venv).

### 3. Настройте источники (опционально)

Отредактируйте файл `input.txt` — добавьте или удалите URL-адреса, откуда скрипт будет загружать списки прокси. По умолчанию уже включены популярные агрегаторы.

### 4. Запустите скрипт

```
python proxy_checker.py
```

После завершения работы вы получите обновлённые файлы:

- `output.txt` — готовые ссылки.
- `history.json` — история проверок.
- `source_stats.json` — статистика по источникам.
- `top30.txt` — 30 лучших прокси по рейтингу.

## Структура репозитория

| Файл | Описание |
|---|---|
| `proxy_checker.py` | Основной скрипт (точка входа). |
| `config.yaml` | Файл конфигурации (зарезервирован для будущих настроек). |
| `input.txt` | Список URL-источников для сбора прокси. |
| `history.json` | Полная история всех проверок с метриками. |
| `output.txt` | Сгенерированные VLESS-ссылки (готовы к импорту). |
| `source_stats.json` | Статистика по каждому источнику (количество, успешность). |
| `top30.txt` | Топ-30 прокси с наивысшим рейтингом. |
| `requirements.txt` | Зависимости Python. |

## Формат данных (пример из `history.json`)

Каждая запись в `history.json` содержит подробную информацию о прокси:

```
{
  "protocol": "vless",
  "host": "5.255.110.147",
  "port": 443,
  "sni": "nll.scroogethebest.com",
  "credential": "e2e1ddaa-1acf-4c4c-a211-92284f6440ed",
  "security": "reality",
  "transport": "tcp",
  "country": "NL",
  "tcp_latency_ms": 83.88,
  "http_latency_ms": 501.99,
  "score": 100,
  "uri": "vless://...",
  "last_alive": true,
  "first_seen": "2026-06-29T06:31:05.368114+00:00",
  "last_seen": "2026-07-01T04:53:44.227967+00:00",
  "appearances": 13
}
```

Ключевые поля:

- `protocol` — протокол (vless, trojan и др.).
- `host` / `port` — адрес и порт.
- `sni` — Server Name Indication (для TLS).
- `security` — тип шифрования (reality, tls).
- `transport` — транспорт (tcp, ws, grpc).
- `country` — страна (по GeoIP).
- `tcp_latency_ms` / `http_latency_ms` — задержки в миллисекундах.
- `score` — общий балл (0–100), чем выше, тем лучше.
- `uri` — готовая ссылка для импорта.
- `last_alive` — статус при последней проверке.

## Пример использования

Запустите скрипт, и через несколько минут вы получите файл `output.txt` с десятками рабочих ссылок. Например, одна из них:

```
vless://e2e1ddaa-1acf-4c4c-a211-92284f6440ed@5.255.110.147:443?type=tcp&security=reality&sni=nll.scroogethebest.com&fp=chrome&pbk=AYQOZFxjvHN-RIzKYEibwFiFsp03cdlWGdCmyFfuNVc&sid=4fa44664f6a566d3&flow=xtls-rprx-vision#🇳🇱 The Netherlands | 🔒100
```

Эту ссылку можно сразу скопировать и вставить в V2Ray / Xray-клиент.
